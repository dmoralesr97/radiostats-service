# Quickstart: Unified Server Dashboard (US-01)

**Date**: 2026-07-04
**Branch**: `001-unified-server-dashboard`

---

## Prerequisites

- Java 21 (JDK)
- Gradle 8.x (wrapper included: `./gradlew`)
- At least one running Icecast 2.x or Shoutcast server reachable from your machine
  (or use the WireMock stub described in §3)

---

## 1. Build & Test

```bash
# Full build with all checks (includes JaCoCo coverage gate)
./gradlew clean check

# Tests only
./gradlew test

# Coverage report only (generates build/reports/jacoco/test/html/index.html)
./gradlew jacocoTestReport
```

The build FAILS if:
- Any unit, integration, or functional (Cucumber) test fails.
- Per-class line coverage drops below 80% for `domain`, `application`, or `adapter` packages.
- Global instruction coverage falls below 80%.

---

## 2. Run the Application

```bash
./gradlew bootRun
```

The service starts on `http://localhost:8080`.

### Key endpoints (once a server is configured)

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/v1/servers/dashboard` | Unified dashboard (live poll) |
| GET | `/api/v1/servers` | List all configured servers |
| POST | `/api/v1/servers` | Register a new streaming server |
| GET | `/dashboard` | HTML dashboard (polls API every 60 s via JS) |

## 3. Database Migrations & Seed Data

Schema and seed data are managed by Flyway. Migrations run automatically at startup.

```text
src/main/resources/db/migration/
├── V1__create_streaming_server_table.sql   # Creates the streaming_server table
└── V2__insert_sample_servers.sql           # Seeds 3 example servers for local dev
```

On first `./gradlew bootRun` Flyway applies V1 (DDL) then V2 (seed data). You can verify:

```bash
# List applied migrations (H2 console at http://localhost:8080/h2-console)
# JDBC URL: jdbc:h2:mem:testdb  User: sa  Password: (empty)
SELECT * FROM flyway_schema_history;
SELECT * FROM streaming_server;
```

The three seed servers (Main Icecast, Backup Shoutcast v2, Legacy Shoutcast v1) appear in
the dashboard immediately after startup. They will show as UNAVAILABLE until real or
WireMock-stubbed endpoints are running.

**Production note**: V2 seed data is excluded in production by setting a different Flyway
location in `src/main/resources/application-prod.yaml`:

```yaml
spring:
  flyway:
    locations: classpath:db/migration/schema   # only V1 DDL; no seed rows
```

The `db/migration/schema/` subdirectory contains only V1. V2 (seed) is in `db/migration/`
only, so it is never applied with the `schema/` path override.

## 4. Configure a Streaming Server (first run)

The seed data in V2 covers the common server types. To register an additional server via
the REST API:

```bash
curl -X POST http://localhost:8080/api/v1/servers \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Main Icecast Stream",
    "serverType": "ICECAST",
    "host": "stream.example.com",
    "port": 8000
  }'
```

For a server requiring credentials, set the environment variable referenced by
`credentialsRef` before starting the application:

```bash
export STREAM_SERVER_1_CREDS="admin:secret"
./gradlew bootRun
```

## 5. Use WireMock Stubs in Tests (no real server needed)

Integration and functional tests use WireMock to stub Icecast / Shoutcast HTTP calls.
The stubs are configured in `src/test/resources/wiremock/` and started automatically by
the test Spring context via `@WireMockTest`.

To run only Cucumber acceptance tests:

```bash
./gradlew test --tests "*.CucumberRunnerTest"
```

Cucumber feature files are at:
```
src/test/resources/features/
├── unified_status_view.feature
├── visual_differentiation.feature
└── auto_refresh.feature
```

---

## 6. OpenAPI Contract & Code Generation

The OpenAPI contract is the source of truth:
```
src/main/resources/openapi/radiostats-api.yaml
```

Server-side stubs are generated by:
```bash
./gradlew openApiGenerate
```

Generated code lands in `build/generated/openapi/` and is included in the compile
classpath automatically. Do **not** edit generated files directly.

## 7. Architecture Compliance Check

ArchUnit tests run as part of `./gradlew test`. To run them in isolation:

```bash
./gradlew test --tests "*.CleanArchitectureTest"
```

The test suite enforces:
- Domain packages have zero Spring / Jakarta dependencies.
- Adapter packages do not cross-reference each other directly.
- Use-case implementations depend only on domain ports.
